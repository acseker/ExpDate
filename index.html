Introduction

This project proposes a generalized framework for recognizing and understanding expiration date of the products. The proposed framework can handle challenging expiration date cases and distinguish 13 different date formats shown in Table 1. Moreover, it can accurately detect and recognize expiration dates even when the input image contains multiple dates. It generates bounding boxes with predictions in the image. Figure 1 shows the results of recognition and understanding expiration date.

Executable Files

We've released executable files of the proposed framework for both windows and ubuntu operating systems. To see the detection and recognition results separately, we divided the proposed framework into two executable files as ExpDateDetection.exe for detecting expiration date and ExpDateRecognition.exe for recognizing and understanding the detected expiration date. Anyone who would like to test their own images can use the following instructions.

Testing Images

Step 1: Create a folder called images_det, that contains the test images.
Step 2: Place the images_det folder and all .exe files in the same directory.
Step 3: Run the following commands for detecting expiration dates. 

# for windows
cd path/to/exe
ExpDateDetection.exe

# for ubuntu
cd path/to/exe
./ExpDateDetection

After running ExpDateDetection.exe file, a folder called results_det that contains detected expiration dates and a folder called images_rec that contains the cropped regions of detected expiration dates, will be created in the same directory as exe files.
Step 4: Run the following commands for recognizing expiration dates. 

# for windows
cd path/to/exe
ExpDateRecognition.exe

# for ubuntu
cd path/to/exe
./ExpDateRecognition

After running ExpDateRecognition.exe file, a folder called results_rec will be created in the same directory as exe files. It contains recognized expiration dates in a .txt file and day, month, and year detection results.

Testing Webcam

We also provide an executable file in which you can use your webcam for recognizing and understanding the expiration date in real-time. You can use the following instructions to use your webcam. 

# for windows
cd path/to/exe
webcam.exe

# for ubuntu
cd path/to/exe
./webcam

ExpDate: Expiration Date Dataset

In the lack of a publicly available dataset, we create a novel dataset, ExpDate, for expiration date detection and recognition. It contains real-world expiration date images captured from food, beverage, and medical products. It also includes challenging cases with various date scales, different date fonts, reflection, complex backgrounds, etc. Figure 2 visualizes some images in ExpDate.


Details

ExpDate contains a total of 1767 images which includes near-horizontal expiration dates. We split it into training and test sets consisting of 1102 training and 665 test images, respectively. Additionally, the test set contains 1956 day, month, and year components, unlike the training set. The width and height of the images vary in the range of [452, 2298] pixels, respectively. The dates are between the years 2018 and 2026. ExpDate has various types of expiration date fonts, including the dot-matrix font. It also includes images with multiple dates on the products (i.e., expiration and production dates). ExpDate includes 13 different expiration date formats shown in Table 1.

Annotation

Each date in the images is annotated with class, bounding box coordinates, date transcription, width, and height of the image in the training set. The test set contains the annotations of the day, month, and year components in addition to date annotation, unlike the training set. Each component is annotated with class, bounding box, and transcription. We utilize the top-left and bottom-right corner coordinates of the bounding boxes.
